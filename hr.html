FAQ on Objective C

Just to save your time on asking the same questions over and over again:
  • Protocol is an abstract interface without implementation that could be used to tell others that some class has some specific methods (conforms to the protocol).
  • Category is a set of methods that extends some existing class. As there're no way to declare protected and private methods in Objective C, category could be used to emulate protected mechanism by declaring methods in a separate header file. Extensions in turn are nameless categories that allow you to change main implementation of the class (override it without creating a subclass). Unlike categories, they allow you to add extra instance variables. Extensions also can be used to emulate private methods.
  • Block is an anonymous function that may contain snapshot of variables stored in stack or heap. Good for callbacks and any other cases where it's useful to have captured state of data for a moment the block was created. Blocks retain objects that are used in their scope.
  • Retain loop is a situation where two (or more) objects are retaining each other thus preventing everybody from been deallocated. For example, passing to some method of self a block that uses self inside will result in a retain loop as block will retain self from within a block and self will retain block. Could be solved both with __weak (in ARC) declaration of a pointer to self or by nillifying of the block after execution.
  • Method swizzling is a way to intercept specific class method and replace it with some other method. It is achieved by creating a category that implements a class method +load, which replaces replaced method implementation with a new one in a dispatch_once block. See more details on the matter here.
  • The difference between dispatch_async, NSOperationQueue and NSThread. NSThread is historically a pre-GCD (Grand Central Dispatch) concept that allows to manually create and execute threads. By contrast GCD automatically creates and maintains four concurrent dispatch queues (three prior to iOS 5) optimizing load on all available processor cores. Unlike GCD, active NSThreads can actually compete for processor time (say, if there're 100 threads running simultaneously) decreasing overal effectiveness. NSOperation is built on top of GCD and gives you a higher level of abstraction and some additional features like bandwidth-constrained queues that only run N-operations at a time as well as establishing dependencies between operations. Summing up, for simple cases it's fine to use pure GCD, NSOperationQueues are useful for complex parallel processing and it's recommended to use any of them over NSThread.
